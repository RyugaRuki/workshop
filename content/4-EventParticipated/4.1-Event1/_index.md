# FCAJ Community Day – May/09/2026

## Event Purpose

The FCAJ Community Day was organized to connect cloud enthusiasts, students, and professionals through technical sharing sessions and career discussions. The event focused on strengthening AWS fundamentals, exploring AI applications, and encouraging participants to develop both technical expertise and professional soft skills.

---

## Speakers

- **Huynh Hoang Long** – *Building Sustainable Learning Habits*
- **Nguyen Tuan Thinh** – *Prompt Engineering for Large Language Models*
- **Anh Khang** – *Preparing Fresh Graduates for the AI Era*
- **Ms. Anh Thao** – *Applying the BMAD Method to AI-Assisted Software Development*

---

## Key Takeaways

### 1. Developing Long-Term Learning Habits

One of the most interesting topics explained why social media is highly addictive while learning technical knowledge often feels difficult. The speaker introduced the concept of using small achievements and rewards to maintain motivation and create consistent study habits.

Important lessons included:

- Break large learning goals into smaller milestones.
- Track daily progress to build discipline.
- Focus on consistency instead of short bursts of motivation.
- Apply gamification techniques to make learning more engaging.

---

### 2. Effective Prompt Engineering

The session demonstrated how structured prompts significantly improve AI-generated responses.

A high-quality prompt should include:

- Role
- Task
- Context
- Input and Output format
- Examples
- Constraints

The speaker also emphasized reducing AI hallucinations by limiting unrelated context and guiding the model with step-by-step reasoning.

A practical demonstration showed how Amazon Bedrock can be integrated with AWS services such as CloudFront, API Gateway, Lambda, DynamoDB, S3, and Cognito to build AI-powered cloud-native applications.

---

### 3. Career Development in the AI Era

Rather than replacing software engineers, AI is becoming a productivity tool.

Key messages included:

- Strong fundamentals remain more valuable than constantly chasing new technologies.
- Understanding *why* a solution works is more important than simply knowing *how* to implement it.
- Career growth should be evaluated through multiple factors, including knowledge, practical experience, professional network, reputation, and financial benefits.

---

### 4. BMAD Method for AI Development

The BMAD workflow demonstrates that AI produces better software when requirements are clearly defined before implementation.

Recommended workflow:

1. Brainstorm ideas
2. Write Business Requirements
3. Design the system architecture
4. Break work into user stories
5. Generate implementation with AI assistance

The presentation highlighted that well-structured documentation leads to higher-quality AI-generated code while reducing technical debt.

---

## Lessons Learned

After attending the event, I gained several valuable insights:

- Build learning habits through small but consistent improvements.
- Use structured prompt engineering when working with AI.
- Strengthen core computer science and cloud computing knowledge.
- Treat AI as a productivity assistant rather than a replacement for engineering thinking.
- Invest time in software architecture and documentation before writing code.
